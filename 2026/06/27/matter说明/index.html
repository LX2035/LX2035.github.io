matter说明

1、基础文档

1. Matter-1.5-Core-Specification.pdf

  • 核心内容： 交互模型 (Interaction Model)、核心数据模型和通信机制。
    • Matter 的语言和语法： 定义了 Endpoint、Cluster、Attribute 和 Command 的数据结构、编码规则和寻址方式。
    • 交互模型： 详细描述了控制器和设备之间如何发送和接收消息（Read Attribute, Write Attribute, Invoke Command, Subscribe to Event）的流程。
    • 安全和网络基础（部分）： 包含安全会话建立、PASE/CASE 过程以及对底层网络传输的抽象定义。

2. Matter-1.5-Application-Cluster-Specification.pdf

核心内容： 所有标准 Cluster (簇) 的功能定义。

  • 功能蓝图： 详述了每个标准 Cluster（例如 On/Off、Level Control、Door Lock、Temperature Measurement 等）必须包含的属性、命令、事件及其确切的行为和数据类型。
  • 设备间的互操作性： 确保所有厂商的设备实现同一 Cluster 时，其功能行为是一致的。
  • 这是开发者在实现具体功能逻辑时查阅最多的文档。

3. Matter-1.5-Device-Library-Specification.pdf

  • 核心内容： 如何将 Cluster 组合成具体的设备类型。
    • 设备类型定义： 定义了 Matter 支持的所有标准设备类型（如 On/Off Light, Dimmable Light, Color Temperature Light, Thermostat, Contact Sensor 等）。
    • 强制和可选 Cluster： 规定了每种设备类型在其各个 Endpoint 上必须 (Mandatory) 支持哪些 Cluster，以及可以可选 (Optional) 支持哪些 Cluster。
    • 例子： “Dimmable Light”设备类型必须在 Endpoint 1 上实现 On/Off Cluster 和 Level Control Cluster。

4. Matter-1.5-Standard-Namespaces.pdf

  • 核心内容： 统一的命名空间、ID 列表和数据类型。
    • ID 注册表： 提供了所有核心 ID 的列表和范围，例如 Vendor ID (VID)、Product ID (PID) 的分配范围、Cluster ID、Attribute ID、Command ID 等。
    • 数据类型定义： 规范了 Matter 使用的所有标准数据类型（如 $INT8$, $UINT16$, $ENUM$, $Struct$ 等）。
    • 枚举和常量： 定义了协议中使用的各种枚举值和常量，确保通信中的一致性。

2、matter数据模型

此处参考：Matter开发，看这一篇就够了 - iini - 博客园

Node

节点(Node)是一个逻辑上独立的设备，有自己唯一的网络地址。每个Matter设备由一个或多个Node组成。

例如 chip-lighting-app 运行的整个软件实例就是一个 Node，但是一个设备可以运行多个这样的软件实例。

Endpoint

一个Node包含多个Endpoint，每个endpoint是一个逻辑上独立的功能模块。比如门锁，它除了可以包含门锁这个endpoint外，它还可以包含温度传感器这个endpoint。

注意：endpoint 0预留为Matter的utility cluster，而且每个Matter设备都必须强制包含它。

例如chip-lighting-app：

  • Endpoint 0: 根端点 (Basic Information, Operational Credentials, etc.)。
  • Endpoint 1: 照明功能 (On/Off Cluster, Level Control Cluster)。

Cluster

Endpoint由一个或多个cluster组成，cluster可以认为是一个基本功能集，它包含attributes（属性）， commands（命令）和events（事件）三个组件。

Matter定义了两种类型的Cluster：

​ Server：提供Attributes, Commands和Events

​ Client：对Server发起交互（interaction）操作

例如：

​ OnOff Cluster (ID 0x0006)： 负责开关控制。

​ Level Control Cluster (ID 0x0008)： 负责调节亮度或百分比等级。

​ Color Control Cluster (ID 0x0300)： 负责颜色和色温控制。

​ Basic Information Cluster (ID 0x0028)： 负责提供厂商、型号、软件版本等信息。

Attribute

Attribute就是一条条表示物理量或者状态的数据记录，他们保存在设备的存储器中。

例如：

  • 在 OnOff Cluster 中：
    • OnOff 属性 (ID 0x0000)： 存储设备的开关状态 (true 或 false)
  • 在 Level Control Cluster 中：
    • CurrentLevel 属性 (ID 0x0000)： 存储设备当前的亮度等级 (值范围 0-254)

Command

Command就是下文所说的action，用来触发server的特定行为。

例如在Onoff Cluster中：

​ On 命令 (ID 0x01)： 告诉设备“请开灯”。

​ Off 命令 (ID 0x00)： 告诉设备“请关灯”。

​ Toggle 命令 (ID 0x02)： 告诉设备“切换当前状态”。

Event

​ Event其实是一种特殊的attribute，它用来更新设备的状态，因此你可以把event当成是一种历史数据记录。
